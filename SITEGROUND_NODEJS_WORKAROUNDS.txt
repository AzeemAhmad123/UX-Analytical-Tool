SITEGROUND NODE.JS WORKAROUNDS - ALL POSSIBLE OPTIONS
================================================================================

Based on research, here are ALL possible ways to run Node.js on SiteGround:

================================================================================
OPTION 1: SSH + Manual Node.js Installation (WORKAROUND)
================================================================================

⚠️ WARNING: Not officially supported, may be unstable

STEP 1: Enable SSH Access
---------------------------
1. Go to SiteGround → Devs → SSH Keys Manager
2. Generate SSH key or import existing
3. Note your SSH port: 18765 (not standard 22)
4. Connect via SSH:
   Windows: Use PuTTY
   Mac/Linux: ssh USER@HOST -p18765 -i /path/to/key

STEP 2: Install Node.js in User Directory
------------------------------------------
1. SSH into your account
2. Download Node.js binary:
   cd ~
   wget https://nodejs.org/dist/v20.11.0/node-v20.11.0-linux-x64.tar.xz
   tar -xvf node-v20.11.0-linux-x64.tar.xz
   mv node-v20.11.0-linux-x64 nodejs

3. Add to PATH:
   mkdir ~/bin
   cp nodejs/bin/node ~/bin
   cp nodejs/bin/npm ~/bin
   echo 'export PATH=$HOME/bin:$PATH' >> ~/.bashrc
   source ~/.bashrc

4. Verify:
   node --version
   npm --version

STEP 3: Install Dependencies
------------------------------
1. Navigate to your backend:
   cd ~/api.enalyze.123fixit.com/public_html

2. Install dependencies:
   npm install

STEP 4: Run Backend (Temporary)
---------------------------------
1. Run directly:
   node index.js

2. Keep it running:
   - Use screen: screen -S backend
   - Run: node index.js
   - Detach: Ctrl+A then D
   - Reattach: screen -r backend

3. OR use nohup:
   nohup node index.js > backend.log 2>&1 &

STEP 5: Keep Process Alive (Challenging)
-----------------------------------------
⚠️ PROBLEMS:
- Process may die when you disconnect SSH
- Process may be killed by SiteGround (resource limits)
- No automatic restart on server reboot
- Port restrictions (can't use port 3001 directly)

WORKAROUNDS:
1. Use PM2 (if you can install it):
   npm install -g pm2
   pm2 start index.js --name backend
   pm2 save

2. Use cron job to restart (if process dies):
   crontab -e
   Add: */5 * * * * cd ~/api.enalyze.123fixit.com/public_html && node index.js

3. Use .htaccess reverse proxy (if allowed):
   ProxyPass /api http://localhost:3001/api
   ProxyPassReverse /api http://localhost:3001/api

OPTION 2: Check Your SiteGround Plan Type
================================================================================

Check what plan you have:

1. Go to SiteGround Dashboard
2. Check your plan type:
   - Shared Hosting → Node.js NOT supported
   - Cloud Hosting → Node.js NOT officially supported
   - VPS → Might work (root access)
   - Dedicated Server → Should work (full control)

If you have VPS or Dedicated:
- You have root access
- Can install Node.js system-wide
- Can use systemd/PM2 properly
- Should work reliably

If you have Shared/Cloud:
- Limited options
- Workarounds are unstable
- Not recommended

OPTION 4: Use PHP Proxy (Hybrid Solution)
================================================================================

Keep backend on Railway/Render, but create PHP proxy on SiteGround:

1. Deploy backend to Railway.app (free)
2. Create PHP proxy file on SiteGround:
   api.enalyze.123fixit.com/public_html/api-proxy.php

3. PHP proxy forwards requests to Railway backend
4. Frontend calls SiteGround PHP proxy
5. PHP proxy calls Railway backend
6. Returns response to frontend

This way:
- Frontend stays on SiteGround
- Backend on Railway (free)
- PHP proxy bridges them (runs on SiteGround)
