STEP-BY-STEP INSTRUCTIONS
================================================================================

STEP 1: Upload package.json (IMPORTANT!)
-----------------------------------------

Your files are in: api.enalyze.123fixit.com/public_html/

You need to upload package.json file:
1. Go to File Manager ‚Üí api.enalyze.123fixit.com/public_html
2. Upload: backend/package.json from your computer
3. This file tells Node.js what dependencies to install

STEP 2: Create Node.js App in SiteGround
------------------------------------------

1. In SiteGround Dashboard, click "Devs" in the left sidebar
   (You should see it in the navigation menu)

2. Look for one of these options:
   - "Node.js"
   - "Application Manager"
   - "Node.js Apps"
   - "Applications"

3. Click "Create Node.js App" or "Add Application" or "New App"

4. Fill in the form with these settings:

   App Name: uxcam-backend
   (or any name you like)

   Subdomain: api.enalyze.123fixit.com
   (Select from dropdown or type it)

   Node.js Version: 20.x
   (or latest available - your code needs Node 20+)

   App Root: /public_html
   (This is where your files are)

   Start Command: node index.js
   (This starts your backend server)

5. Click "Create" or "Save"

STEP 3: Install Dependencies
-----------------------------

After creating the app, SiteGround should automatically:
- Run `npm install` to install dependencies from package.json
- If not automatic, you may need to click "Install Dependencies" button

STEP 5: Start the App
----------------------

1. Make sure the app status is "Running" or "Started"
2. If it says "Stopped", click "Start" button
3. Wait for it to start (may take 30-60 seconds)

STEP 6: Test the Backend
-------------------------

1. Visit: https://api.enalyze.123fixit.com/health
2. Should see: {"status":"ok","timestamp":"..."}

If you see an error:
- Check app is started
- Check start command is: node index.js
- Check environment variables are set
- Check logs in Node.js app settings

================================================================================
ALTERNATIVE: If "Devs" Section Not Available
================================================================================

Some SiteGround plans may not have Node.js support. In that case:

OPTION 1: Use cPanel (if available)
------------------------------------
1. Go to cPanel
2. Look for "Application Manager" or "Node.js Selector"
3. Follow similar steps

OPTION 2: Contact SiteGround Support
-------------------------------------
Ask them to enable Node.js for your account or help you set it up.

OPTION 3: Use Different Hosting
---------------------------------
If Node.js is not available, consider:
- Railway.app (free tier available)
- Render.com (free tier available)
- Vercel (for serverless)
- Heroku (paid)
