================================================================================
WHY CREATE A NODE.JS APP? - EXPLANATION
================================================================================

üìå WHY WE NEED TO CREATE A NODE.JS APP:
----------------------------------------

Your backend files (index.js, routes, services) are JavaScript files that need
to RUN on the server, not just be served as static files.

Think of it like this:
- Frontend files (HTML, CSS, JS) = Static files ‚Üí Just need to be served
- Backend files (Node.js) = Application code ‚Üí Needs to RUN and process requests

Without creating a Node.js app:
‚ùå Your files just sit there doing nothing
‚ùå No server running to handle API requests
‚ùå https://api.enalyze.123fixit.com won't work

With a Node.js app:
‚úÖ Server runs your code continuously
‚úÖ Handles API requests (GET, POST, etc.)
‚úÖ Connects to database
‚úÖ Processes requests and returns responses

================================================================================
HOW TO CREATE NODE.JS APP IN SITEGROUND
================================================================================

METHOD 1: Using SiteGround Devs Section (Recommended)
------------------------------------------------------

1. In SiteGround Dashboard, click "Devs" in the left menu
2. Look for "Node.js" or "Application Manager" option
3. Click "Create Node.js App" or "Add Application"

4. Fill in the form:
   - App Name: uxcam-backend (or any name)
   - Subdomain: api.enalyze.123fixit.com
   - Node.js Version: 20.x (or latest available)
   - App Root: /public_html (or leave default)
   - Start Command: node dist/index.js
     (OR if files are directly in public_html: node index.js)

5. Click "Create" or "Save"

6. The app will start automatically

METHOD 2: Using Application Manager (Alternative)
---------------------------------------------------

1. Go to: Site ‚Üí Application Manager
2. Click "Create Application"
3. Select "Node.js"
4. Follow similar steps as Method 1
